## Description

> Imported from gh#112 — https://github.com/awizemann/scarf/issues/112

**Describe the bug**
After successfully getting the key based authentication set up, the app is running, but I get the error “Chat connection Failed - Couldn’t save model+provider to config.yaml.”
Memory files do not open.

This likely is an issue to do with being installed in Docker. The file path for memory, etc. are different in Docker than in bare metal installs:

'~.hermes/config.yaml' not found on [redacted x.x.x.x]. Once Hermes is configured on this host, Settings will light up.

**To Reproduce**
Steps to reproduce the behavior:
1. Open the Scarf iOS app and successfully set up the key (note: Docker users cannot be inside the “docker shell” to enter the key from the app they must exit first)
2. Attempt to enter the provider and model.
3. See error

**Expected behavior**
Scarf loads existing settings and works

If applicable, add screenshots.
