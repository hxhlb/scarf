## Source of truth (don't reinvent)

The release PROCESS is already documented — this plan only covers the per-release prep + a
checklist. Read these first:
- **Process:** `wiki/Release-Process.md` (public summary) — the script is the source of truth.
- **Ops note:** `.memory/ops/Build and Release Workflow.md` (commands, prereqs, gotchas).
- **Constraints:** `.memory/ops/Sparkle Key Recovery.md`, `.memory/ops/Releases under iCloud Drive.md`.

**The cut itself:** `./scripts/release.sh 2.10.3` — run by Alan on the release machine. It needs
the Developer ID cert, the `scarf-notary` keychain profile, and the **per-machine Sparkle EdDSA
key** (release.sh preflights it against `Info.plist:SUPublicEDKey` and aborts on mismatch). NEVER
run manual `xcodebuild archive` / `notarytool` / `gh release create` steps.

## Cut steps (Alan, on the release machine)

1. **Push main** — `git push origin main` (local main is 18 commits ahead of origin/main after
   today's merge; the release is cut from main).
2. **Verify the Sparkle key** is the production one on this machine (release.sh preflights, but
   know the recovery runbook: `.memory/ops/Sparkle Key Recovery.md`).
3. **Cut:** `./scripts/release.sh 2.10.3` (or `--draft` first to stage, then promote per
   `wiki/Release-Process.md` → "Promotion (draft → real)").
4. The script bumps `MARKETING_VERSION` 2.10.2 → 2.10.3 across the project, archives Universal +
   ARM64, signs/notarizes/staples, EdDSA-signs + pushes the appcast to gh-pages, tags `v2.10.3`,
   and creates the GitHub release with both zips + the notes as the body.
5. Confirm Sparkle offers the update (Scarf → Check for Updates) and the appcast at
   `https://awizemann.github.io/scarf/appcast.xml` carries the new entry.
