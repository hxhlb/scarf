## Highlights by major

**2.12.0** — Coordinated catch-up to **Hermes v0.17.0 (2026.6.19)** (the largest Hermes release yet — Scarf needed only a focused slice), bundled with a **remote-chat performance fix**: typing into a session on an SSH host no longer lags or spikes CPU now that watcher-driven reads (credential preflight, sessions list, platforms, projects) run off the main thread instead of doing synchronous SSH round-trips per message ([#119](https://github.com/awizemann/scarf/issues/119)). The v0.17 audit found the upstream surface — `state.db` schema, ACP wire protocol, CLI verbs, config keys, model catalog — **entirely stable** (zero mandatory changes), so the release is a feature catch-up plus a cluster of **pre-existing Scarf bugs the audit exposed**: four broken Health/Settings actions (`hermes audit`→`security audit`; `migrate xai`→`--apply` so it stops claiming a no-op succeeded; browser-setup `--assume-yes`→`--yes`; the no-op WhatsApp recipient allowlist removed), and the **Curator "Prune" mislabel** — the action claimed to permanently delete archived skills from disk, but the real verb archives *idle active* skills reversibly (and Scarf invoked it wrong, so it hung), now rebuilt as **"Archive idle skills"** with a 30/60/90/180-day threshold, an idle-skill preview, and a reversible confirm. New v0.17 surfaces, all capability-gated behind `isV017OrLater`: **WhatsApp Business Cloud API** and **SimpleX** setup forms, **Telegram** rich-messages + online/offline status toggles, an opt-in **curator consolidation** toggle (v0.17 turned the LLM skill-merge pass off by default), and a **max-concurrent-sessions** cap. An adversarial fresh-eyes audit of the branch caught a real pre-release bug (built-in gateway platforms need an explicit `enabled` write, unlike auto-enabling plugins). Deferred for later: iMessage via Photon (the protocol is still changing upstream), the per-MCP-server keepalive knob, and a pre-existing Google Chat allowlist fix. Pre-v0.17 hosts render the v2.11.0 surface byte-identical. 642 ScarfCore tests.

**2.11.0** — Coordinated catch-up to **Hermes v0.16.0 (2026.6.5)**, led by the first `state.db` schema change since v0.11: the `messages` table gained an `active` column so Hermes can **soft-delete rewound messages** on `/undo`. Scarf now filters `AND active = 1` on all six message-read paths, schema-gated (local `PRAGMA` / remote preflight) so pre-v0.16 DBs don't throw "no such column" — a rewound chat shows what the agent actually sees instead of discarded turns. **Live session titles** arrive via the new ACP `session_info_update` notification (Mac sidebar updates a regenerated title in place; there is no ACP set-title RPC — that stays CLI/gateway). A **"rewound ×N" badge** surfaces the new `sessions.rewind_count` column (schema-detected, read by name) in the chat sidebar, Sessions table, and dashboard. **Spotify** stopped being a skill and became a built-in tool + native plugin in v0.16, so its sign-in affordance moved to the Plugins view on v0.16 hosts (SkillsView path kept for older ones). Two silent **gateway-config bugs** found while verifying writes against the live source were fixed: platform allowlists were written under `gateway.platforms.<p>.*` but Hermes enforces them top-level (`slack.allowed_channels` / `telegram.allowed_chats` / `matrix.allowed_rooms` / `dingtalk.allowed_chats` — DingTalk also corrected from `allowed_rooms`), so Scarf-set allowlists never applied; and the cross-profile gateway digest assumed a `hermes gateway list --json` flag that doesn't exist (text output only), so it silently hid — now parsed from the text table. **v0.16 quick wins:** an "Optimize sessions database" Health action (`hermes sessions optimize`), a Kanban "Goal · N" pill (`goal_mode`/`goal_max_turns`), and capability-gated session rename. **Config-shape corrections** verified against live v0.16: OpenRouter `response_cache` is a scalar bool (so a *disable* takes effect, vs. the old nested dict Hermes always read as on); the MCP SSE add path matches the real `hermes mcp add` flags (URL on add, `transport: sse` stamped into YAML); the dead Kanban `verify` verb was removed (Reject routes through `comment` + `archive`); `curator list-archived` text is parsed directly (no phantom `--json`); ACP `/goal` + `/subgoal` were de-advertised as gateway-only. AWS Bedrock is registered as an overlay-only provider; the other v0.16 providers self-surface from the catalog and new models (deepseek-v4-flash, MiniMax-M3, qwen3.7-plus, gemini-3.5-flash) flow automatically (upstream catalog TTL now hourly). New capability flags (`hasSessionsRename`, `hasSessionsOptimize`, `hasKanbanGoalMode`, `hasInsightsCommand`, `hasDashboardCommand`) + the `isV016OrLater` predicate gate every surface; pre-v0.16 hosts render the v2.10.3 layout byte-identical. 628 ScarfCore tests. Deferred (additive, P2): sessions-optimize-as-maintenance, the `insights`/`dashboard` CLI affordances, and Kanban run-terminate (REST-only in v0.16).

**2.10.0** — Coordinated catch-up to Hermes v0.15.0 ("The Velocity Release"). **OpenAI lands as a first-class provider** (wire ID `openai-api`, distinct from the OpenAI Codex runtime; bare `openai` stays a Hermes alias to OpenRouter so it's intentionally NOT registered). **Krea image-generation models** (`krea-2-medium`, `krea-2-large`) join the image-gen list. **xAI May-15 model retirement** maps retired Grok IDs (`grok-4-0709`, `grok-4-fast-*`, `grok-4-1-fast-*`, `grok-code-fast-1`, `grok-3` → `grok-4.3`; `grok-imagine-image-pro` → `grok-imagine-image-quality`) forward so a stored retired model still resolves, while **Vercel AI Gateway (provider) + Vercel Sandbox (terminal backend) are removed** — deleted upstream, dropped from Scarf's picker + terminal-backend list. **xAI Web Search** is a new `web_tools.search.backend: xai` option reusing your Grok OAuth / `XAI_API_KEY`. **ntfy** lands as the **23rd gateway platform** (push notifications via a topic URL, no account; `platforms.ntfy.extra.{topic,server,publish_topic,token,markdown}`) alongside new per-platform flags (Telegram `disable_topic_auto_rename` + `ignore_root_dm`, Discord `allow_any_attachment`, Signal group-only `require_mention`). **xAI TTS `auto_speech_tags`** is an opt-in toggle (default off). The **Kanban v0.15 maturation wave**: server-side `--sort` (priority / created / status / assignee / title / updated) as a board-header picker; Promote (`todo`/`blocked` → `ready`), Schedule / Park, and Delete-permanently (`archive --rm`) card actions; new **Scheduled** + **Review** columns (collapse when empty); per-task worktree `--branch` on create + read-only `model_override` in the inspector; a **precise chat-scoped board** keyed by the originating ACP `session_id` (replacing the old tenant + time-window approximation) with a "This chat ⇄ All tasks" scope toggle; and `--board` multi-board plumbing in the service layer (board switcher UI + `swarm` create-sheet are follow-ups). **Bitwarden Secrets Manager** gets a new Settings → Secrets tab for `secrets.bitwarden.*` (enabled, `access_token_env` default `BWS_ACCESS_TOKEN`, project_id, override_existing, server_url US/EU/self-hosted, cache_ttl_seconds, auto_install — the bootstrap token lives in `~/.hermes/.env`). **MCP mTLS client certificates** (`client_cert` / `client_key` / `ssl_verify`) surface in the server editor for HTTP + SSE servers, alongside a read-only `hermes mcp catalog` browse sheet. **Skill bundles** get a read-only Bundles tab over `~/.hermes/skill-bundles/*.yaml`. **Per-session edit-approval modes** add a chat-header chip toggling a live session between Default / Accept Edits / Don't Ask via ACP `session/set_mode` (distinct from the global `approvals.mode` / YOLO surface; sensitive paths always still prompt). The **Health view** gains a "Run supply-chain audit" button (`hermes audit`, OSV.dev) + an xAI retired-model warning offering one-click `hermes migrate xai`. New v0.15 capability flags (`hasKanbanSessionFilter`, `hasKanbanV015`, `hasNtfyPlatform`, `hasXAIWebSearchBackend`, `hasXAITTSAutoSpeechTags`, `hasBitwarden`, `hasHermesAudit`, `hasXAIModelRetirement`, `hasMCPClientCerts`, `hasMCPCatalog`, `hasSkillBundles`, `hasSessionEditAutoApproval`, `hasAzureEntraAuth`) + the `isV015OrLater` predicate gate every surface; pre-v0.15 hosts render the v2.9.x layout unchanged. The bulk of v0.15 (the `run_agent.py` refactor, cold-start performance, promptware defense, `session_search` rebuild, the Ink TUI orchestrator, the web dashboard, Docker s6 supervision, the API-server session-control REST) is server-side or other-frontend and benefits Scarf transparently with no Scarf change. Pre-release review fixes ride along: Kanban session-scope correctness (the chat-scoped board no longer drops session tasks the agent created without tagging the project tenant — the precise `--session` filter is authoritative), the scope pill now shows for global chats too, MCP SSL-verify and CA-bundle path are independent controls, and dead time-window heuristic code was removed.

**2.9.2** — Patch fixing two reported bugs in remote chat and session resume. **[#100](https://github.com/awizemann/scarf/issues/100) Remote Chat blocked on a bare `hermes` binary** — the pre-flight gate ran `fileExists("hermes")` which on a remote context executes `test -e hermes` (a filesystem check against the remote cwd, not a `PATH` lookup), so a pipx install resolvable on the login `PATH` still showed "Hermes Not Found" even with 14/14 Remote Diagnostics. `ServerContext.hermesBinaryProbablyResolvable()` now distinguishes a path-shaped binary (keeps the accurate `fileExists` check) from a bare command name (presumed resolvable; authoritative check deferred to the login-shell ACP launch). **[#99](https://github.com/awizemann/scarf/issues/99) Session resume silently dropped context** — `ACPClient.loadSession` treated Hermes's JSON-RPC `result: null` (returned when a session can't be restored into the ACP runtime) as success, so prompts landed in a phantom fresh session. `loadSession` now throws on a null/non-dict result, routing into the existing fallback that creates a fresh session and replays the transcript from `state.db` so the thread reads continuously. 3 new ScarfCore tests.

**2.9.1** — Patch covering [#97](https://github.com/awizemann/scarf/issues/97) — the v2.9.0 mid-chat model switcher + per-project preset binding fired `session/set_model` with the bare model ID, never the colon-prefixed `<provider>:<model>` wire format Hermes's ACP adapter expects. Auto-detectable IDs (e.g. `claude-opus-4-7`) worked; less-obvious slugs (e.g. OpenRouter `<author>/<model>` like `inclusionai/ring-2.6-1t`) fell through to `detect_provider_for_model` and inferred the wrong provider. `ACPClient.setSessionModel` gains an optional `providerID:` parameter (encodes `"<provider>:<model>"` matching `_encode_model_choice`, falls back to bare model when nil); `switchModelPreset` + `applyProjectModelPreset` pass `preset.providerID` through, and "Use global default" reads `config.provider` so reverting lands on the same provider the CLI would. New public `ACPClient.encodeModelChoice(modelID:providerID:)` helper + 6 new `M1ACPTests`. Forward-compatible — Hermes accepts both wire shapes on v0.13+.

**2.9.0** — Coordinated catch-up to Hermes v0.14.0 ("The Foundation Release"). Ships the v0.14 ACP slash commands — **`/subgoal`** (extra success criteria layered onto an active `/goal` loop; forms `<text>` / `remove N` / `clear`; the goal pill gains a `+N` count badge with the full list in the tooltip) plus **`/yolo`**, **`/sessions`**, and **`/codex-runtime`** as new always-available menu items. `/handoff` is intentionally NOT in the slash menu — v0.14 marks it `cli_only` in the command catalog (it hands a session to a messaging platform, not to a different model); mid-chat model switching stays on the existing `session/set_model` RPC. **Two new inference providers** — xAI Grok OAuth (SuperGrok) as a subscription-gated overlay (wire ID `xai-oauth`, base URL `https://api.x.ai/v1`) and NovitaAI as an API-key overlay (wire ID `novita`). **Alibaba Cloud → Qwen Cloud display rename** mirrors Hermes's picker label without changing the wire ID; existing `alibaba` / `alibaba-cloud` config keys keep working. **Two new gateway platforms** — LINE Messaging API (21st) as a first-class native adapter and SimpleX Chat (22nd) talking to a local `simplex-chat` daemon over WebSocket. **Two new web-search backends** — Brave Search (free tier; wire ID `brave-free`; honors `BRAVE_SEARCH_API_KEY`) and DuckDuckGo (DDGS; wire ID `ddgs`; anonymous, uses the lazy-installed `ddgs` Python package). The Scarf list also got a wire-accuracy pass — stale `duckduckgo` / `brave` / `you` entries that never matched any real Hermes backend ID were replaced with the canonical v0.13 set (`exa`, `parallel`, `firecrawl`, `tavily`, `searxng`). **Hermes Proxy** lands as a new Configure → Hermes Proxy sidebar destination wrapping `hermes proxy start --provider <p> --host 127.0.0.1 --port 8645`. The new [`HermesProxyService`](https://github.com/awizemann/scarf/blob/main/scarf/scarf/Core/Services/HermesProxyService.swift) (`@MainActor @Observable`) owns the long-running child process, drains stderr into a 200-line capped log buffer, advertises `isRunning` + `endpoint` + `routedProvider`, and probes `hermes proxy providers` for the adapter list. v0.14 ships with `nous` as the only adapter; the picker auto-refreshes when more land. Point any OpenAI-compatible client (Codex CLI, Aider, Cline, VS Code Continue) at `http://127.0.0.1:8645/v1` and any bearer token works — the proxy attaches your real credential. Local server only in v2.9; SSH-deployed hosts get an explanatory notice (port-forward wiring is deferred). **MCP `supports_parallel_tool_calls`** surfaces as a tri-state segmented picker on the server editor ("Default (Hermes decides)" / "Enabled" / "Disabled"); writer either inserts the scalar or drops the key entirely. **ACP `--setup-browser`** wires to a "Set up browser tools" button on the Health view header — runs `hermes acp --setup-browser --assume-yes` off MainActor with a 10-minute timeout (chromium download + playwright install) and an inline status strip. **YOLO mode warning chip** renders in the chat header when `approvals.mode = yolo` and the host advertises `hasYOLOWarning`. Smaller v0.14 surfaces: **`terminal.docker_extra_args`** (extra flags to `docker run`, comma-separated input), **`display.timestamps`** (per-message timestamps in the TUI — also fixed a pre-existing v0.13 oversight where `display.language` was in the model but the YAML parser wasn't reading it back), **Cron `deliver=all`** (fan-out routing intent surfaced in the placeholder/helper text), **Discord channel-history backfill** toggle (default-on to match v0.14), **plugin `tool_override` badge** on PluginsView. **24 new capability flags** + the `isV014OrLater` predicate gate every surface; pre-v0.14 hosts render the v2.8.0 layout byte-identical. **Bundled fixes since v2.8.0:** per-project model presets + mid-chat switcher via `session/set_model` (capability-gated on `hasACPSetSessionModel` from v0.13), Kanban toolset enable YAML write fix, MetricKit crash + hang diagnostics on iOS, JSON read caps in session-attribution + project-dashboard, scroll crash + background-lifecycle hardening, selectable text across paragraphs ([#93](https://github.com/awizemann/scarf/issues/93)), process pipe draining for large Kanban output ([#95](https://github.com/awizemann/scarf/issues/95)), transcript render perf (cron filter + two-stage load-earlier), iOS slash-command parity, `/steer` honesty pre-session.

## After a release

When `scripts/release.sh <version>` completes a full (non-draft) release, this page should be updated:

1. Add a row at the top of the table.
2. Bump the **Latest release** line on [Home](Home).
3. Run `./scripts/wiki.sh commit "docs: index v<version> release"` then `push`.

This is one of the [wiki update triggers](Wiki-Maintenance) that future Claude Code sessions will follow automatically.
