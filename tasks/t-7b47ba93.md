## Description

FIXED + committed on `fix/remote-sync-io-on-main` (3 commits: bd2f55b off-main fix, b0eae76 import cleanup, 2dbee49 second-audit fixes), NOT pushed — merges into the next release with Hermes v0.17.

ROOT CAUSE: file-watcher `.onChange(fileWatcher.lastChangeDate)` ticks (fire per persisted message during a stream) ran synchronous remote (scp/SSH) reads on the MainActor → typing lag on remote hosts.

FIXED SITES (5): ChatViewModel credential preflight (off-main + debounced scheduleCredentialPreflightRefresh); PlatformsViewModel.load (off-main gatewayState + cached configuredPlatforms — kills the per-render hasConfigBlock config.yaml/.env reads from the view body); ProjectsViewModel (ScarfCore, made @MainActor; off-main reload()); HermesEnvService.load/stripEnvQuotes nonisolated; SessionsViewModel.computeStats stat() folded into load()'s off-main batch. + 2 pre-existing missing-import warnings.

TWO FRESH-EYES AUDITS (adversarial subagents):
- Audit 1 caught an out-of-order race: the first async cut dropped the codebase's cancel/coalesce protection → an older read could clobber fresh state. Fixed with cancel-prior + recency guards.
- Audit 2 (2 independent reviewers) caught 3 more, all fixed: (a) SessionsViewModel.computeStats did a sync stat() on main per tick — MISSED by the first sweep because load() is async but the stat *inside* it wasn't; (b) ProjectsViewModel.reload's dashboard write sat behind a 2nd suspension gated on Task.isCancelled (not recency) → could latch a stale dashboard — replaced with a monotonic generation token gating BOTH the registry and dashboard commits; (c) PlatformsViewModel.restartGateway captured self strongly across a 30s blocking call → [weak self]. Reviewers also CONFIRMED: hasConfigBlock→computeConfiguredPlatforms parse equivalence (incl. CRLF quirk), reload()/load() equivalence, the other 9 watcher handlers already clean, and the iOS hot path clean (no fileWatcher, no preflight banner, doesn't use the shared ProjectsViewModel). Note: ScarfCore is Swift-5 language mode, so the compiler does NOT enforce strict concurrency there — the recency logic was verified by review, not the compiler.

VERIFIED (final): macOS build 0 warnings; ScarfCore swift test 630/630 (the 1 transient miss = known t-aud32 flake, passes in isolation); ScarfCore compiles for iphonesimulator SDK 0 warnings; app Projects/Sessions test suites pass.
